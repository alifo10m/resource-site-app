# resource-site-app

一个用于归档和发布多个独立 HTML 页面的仓库。  
本项目不针对任何特定域名或具体网站，仅作为静态 HTML 资源的集合与展示用途。

## 项目简介

`resource-site-app` 主要用于存放各类独立的 HTML 页面，这些页面可以是工具界面、信息展示页、原型演示或其他类型的静态内容。  
仓库本身不提供动态服务或后端逻辑，所有页面均为纯前端 HTML，可直接在浏览器中打开。

## 目录说明

```
resource-site-app/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 入口页面（可选）
└── README.md       # 本文件
```

- **pages/**：每个子文件夹或文件代表一个独立页面，命名清晰便于识别。
- **assets/**：存放页面间共享的样式、脚本或媒体文件，避免重复引用。
- **index.html**：如果存在，可作为仓库的导航或说明首页。

## 页面归档说明

- 每个 HTML 页面应保持完整自包含，依赖的本地资源需放在 `assets/` 目录下。
- 页面内容应不包含外部追踪代码、恶意脚本或违反平台政策的信息。
- 归档的页面可能不定期更新或清理，建议使用者自行备份重要内容。

## 维护说明

- 本仓库由维护者不定期更新，新增或删除页面会通过 commit 记录体现。
- 欢迎通过 Issues 提出关于页面组织或资源管理的建议，但不接受针对具体网站的功能请求。
- 提交 Pull Request 前请确保页面格式规范，且不引入外部敏感链接。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可将仓库部署到任何静态托管服务（如 GitHub Pages）进行在线访问。
